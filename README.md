# AI Coding Agent System Prompt

> 为 AI 编程助手设计的持久化记忆体系与行为规范


## 📖 这是什么？

这是一份专为 AI 代码智能体（Coding Agent）设计的**完整系统提示词**，核心目标是：

- **让 AI 拥有项目长期记忆**：通过 `Memory.md` 知识库驱动开发，避免每次对话都从零开始
- **让 AI 成为严谨的工程师**：只改该改的代码，不写冗余代码，改完验证通过才算完
- **让 AI 行为可预期、可追溯**：每一行改动都能追溯到具体需求

**它不是一份泛泛的"思考原则"，而是一套可落地、可重复使用的行为规范与知识管理框架。**


## 🎯 核心特色

### 1. Memory.md 知识库驱动（原创扩展）

这是本提示词在社区通用准则基础上的**原创扩展**。要求 AI 在每个项目中维护一份 `memory.md` 文件，记录：

- 项目概述与技术栈
- 架构设计与模块说明
- 编码规范与命名约定
- 重要决策记录（ADR）
- 开发进度与详细实现记录
- 常见问题与解决方案

**效果**：AI 每次接手项目，先读 `memory.md`，秒变"老员工"。

### 2. 四大行为准则

以下准则的核心理念源自 **Andrej Karpathy**（OpenAI 联合创始人、前特斯拉 AI 总监）对 LLM 编程 Agent 系统性失败模式的观察，经社区传播形成通用规范。

| 准则 | 说明 |
|------|------|
| **编码前先思考** | 不确定就停下来提问，不假设、不猜测 |
| **极简优先** | 只写最少代码，不做过度设计，200行能缩到50行就缩 |
| **手术式修改** | 只动必须改的部分，不重构没坏的东西，保守删除 |
| **目标驱动** | 定义可验证的成功标准，闭环执行，改完验证通过才算完 |

### 3. 完整的任务处理流程

- **理解与规划** → **执行与实现** → **验证** → **交付与清理**
- 每个阶段都有明确的输入输出和检查点

### 4. 终端编码规范

解决 Windows 下 CMD / PowerShell 中文乱码问题，要求执行任何命令前先 `chcp 65001`。

## 🚀 适用场景

本提示词适用于以下 AI 编程工具：

| 工具 | 如何使用 |
|------|---------|
| **Trae** | 设置 → 规则 → 创建 `user_rules.md` 或 `project_rules.md`，粘贴内容，选择"始终生效" |
| **Cursor** | Settings → General → Rules for AI，粘贴内容保存 |
| **Claude Projects** | Project Instructions 中粘贴本提示词 |
| **Dify / Coze** | 创建 Agent → 系统提示词中粘贴 |
| **OpenAI GPTs** | Configure → Instructions 中粘贴 |


## 🛠️ 如何使用

### 快速开始（3 步）

1. 复制 `prompt.md` 的全部内容
2. 粘贴到你使用的 AI 编程工具的"系统提示词"或"规则"设置中
3. AI 将立即按照本规范工作

### 进阶使用：初始化项目的 Memory.md

在项目中首次使用本提示词时，让 AI 执行：

```
请根据本提示词中的 Memory.md 模板，为当前项目创建 memory.md 文件，并填充项目名称、技术栈、结构等信息。
```

之后，AI 会自动维护这份"项目活文档"。
